## 你的审查角度：结构性与概念一致性

与 Codex 互补——不重复 Codex 已覆盖的机械检查。聚焦以下：

### D1: SOP 逻辑自洽性
- 四步流程（事实提取→评分→挑战→终判）之间是否存在逻辑裂缝？
- 各节之间的交叉引用是否正确？有没有死链接？
- 约束和例外之间的优先级是否清晰？（例：什么时候可以跳过 T2 审查？C6 不可行时怎么办？）

### D2: 概念定义质量
- 关键概念（独立性、裸环境、魔鬼代言人、证据层级）的定义是否精确、无歧义？
- 不同读者（首次接触 vs 有经验的用户）能否从 SOP 中建立一致的理解？
- 有没有概念 A 在 §X 和 §Y 中定义不一致的情况？

### D3: Prompt 模板可用性
- 复制粘贴到全新会话后，AI 审查者是否可能产生歧义理解？
- full-review.md 和 adversarial.md 两个模板的边界是否清晰？（什么时候用哪个？）
- 模板中"被审查的材料"占位符之前的指令是否足够自包含？

### D4: 与源材料的一致性
- 从框架 v1.6.4 提取 SOP 的过程中，有没有改变了源材料的核心精神？

### D5: 整体印象
- 作为一个独立产品，这个工具包给人的第一印象是什么？
- 5 分钟上手、目录结构、实战数据、审查案例——这些元素之间的比例和顺序是否合理？
- README 的语气是否在"自信"和"诚实"之间取得了平衡？
