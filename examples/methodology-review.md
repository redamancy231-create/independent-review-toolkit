## 背景

被审查材料：一份约 2,000 字符的实验报告，描述了一个对照实验——检验层级化工作流描述（Flow-as-Node）vs 内容等价的扁平描述，在编码 Agent 工作流理解任务上的表现差异。

结论声明：Δ median F1 = 0.000，3/5 类别天花板效应，标记为 [E-] ceiling-limited（Tier 0 负证据）。

审查目标：验证实验设计、数据分析方法、结论声明的可靠性，以及天花板效应的解释是否合理。

---

## 审查发现

### 步骤一：事实提取（代表性声明摘录）

| 声明 | 置信度 | 类型 | 验证来源 |
|------|--------|------|---------|
| "n=20/臂，GPT-5.5 temp=0" | VERIFIED | FACT | prompt-tdd 项目实验配置文件 |
| "Δ median F1 = 0.000" | CONSISTENT | FACT | 与统计报告一致；审查者无法直接验算 → TOOL-LIMITED |
| "3/5 类别天花板效应" | CONSISTENT | INFERENCE | 框架 §6.3.2 逐类分析 |
| "Tier 0 负证据" | CONSISTENT | INFERENCE | 实验设计为 Tier 0（基础对照），阴性结果本身是有效证据 |
| "ceiling-limited" 定性 | PLAUSIBLE | VALUE | 3/5 类别 F1 > 0.95 支持天花板解释；另 2 类 n=4 限制置信度 |

> 审查者标注 1 项 TOOL-LIMITED：无法访问原始评分脚本以独立复现 F1 计算。

### 步骤二：独立评分

| 维度 | 分数 | 锚定说明 |
|------|------|---------|
| D1 逻辑有效性 | 4/5 | 评推理过程：A/B 臂定义清晰，对照逻辑正确。实验设计本身合理——但结论从"未检测到差异"到"格式效应不存在"的推理跳跃未充分桥接 |
| D2 事实准确性 | 4/5 | 先验证事实：统计报告数值与实验配置一致。temp=0 选择和 n=4 子类的事实准确——但这两个事实的方法学后果（见 D4）未充分讨论 |
| D3 方法适当性 | 4/5 | 评方法选择：对照实验是检验格式效应的适当方法。但 F1 作为唯一指标可能对结构性差异不敏感（评分者推理的局限，非结论判断） |
| D4 完整性 | 3/5 | 评覆盖范围：未讨论 temp=0 对结论可推广性的限制（发现 1）；未逐类报告样本分布和 ceiling 阈值（发现 2）；未讨论层级化描述和扁平描述的信息量是否等价这一关键假设（发现 3）；负面结果中 precision/recall 拆分缺失（发现 4） |
| D5 清晰度 | 4/5 | 评表达质量：结构清晰，[E-] 证据标注体系使用恰当，术语定义明确 |
| D6 洞察深度 | 4/5 | 评洞察水平：阴性结果（Δ=0.000）的诚实报告本身在提示工程文献中稀缺，具有方法论价值。但将阴性结果归因于"天花板效应"的论证链条可进一步加强 |

**综合评分**：3.8/5（D1-D6 算术平均）

### 步骤三：对抗式挑战

**(A) 替代解释** [MAJOR]  
天花板效应是否可能源自评分 rubric 太粗糙（粗粒度 F1 无法区分结构性差异），而非真正的"格式无影响"？→ 针对框架 §6.3.2 F1 为唯一指标的段落。建议补充 precision/recall 拆分或错误类型分布。

**(B) 隐藏假设** [MAJOR]  
实验隐含假设了"层级化描述和扁平描述的信息量等价"——如果层级化描述实际传达了更多结构信息，则实验检验的是信息量效应和格式效应的混合，而非纯格式效应。→ 针对实验设计中 A/B 臂"内容等价"的定义。

**(C) 边界条件** [MODERATE]  
结论的推广边界不够清晰——"GPT-5.5 中文编码任务内"的结论是否适用于：(a) 其他模型后端？(b) 非编码任务？(c) temp > 0 的生成场景？→ 针对框架 §6.3.2 结论段的推广性声明。

**(D) 反例/反证** [MINOR]  
提示工程文献中存在格式效应阳性的报告（如 Chain-of-Thought 格式的显著效果）——这些是否构成 Flow-as-Node 格式效应的反例，还是属于不同机制？建议区分"内容格式"vs"推理格式"。

**(E) 方法论替代** [MODERATE]  
F1 作为单一指标：如果用精确匹配率（Exact Match）或人类评估作为替代指标，结论是否一致？→ 针对框架 §6.3.2 的评估方法选择。

### 挑战后评分调整

- D4 完整性：维持 3/5——挑战 (A)(B)(E) 进一步确认了完整性维度的缺失项，但不改变初始评分级别
- D3 方法适当性：维持 4/5——挑战 (E) 指出了单一指标的局限，但不否定对照实验方法本身

调整理由：挑战未发现步骤二未识别的新致命问题；(A)-(E) 五个方面的发现均在 D4 完整性的初始 3/5 评分中已隐含，挑战使之显式化。维持原评分。记录：经受住 (A)-(E) 挑战，评分不变。

### 步骤四：终判

**判决**：**Minor Revision（小改）**

综合评分 3.8/5。材料在实验设计、方法选择、表达质量和诚实报告方面合格；核心发现（ΔF1=0.000）可信。但存在 4 项完整性方面的改进空间（D4=3/5），均不改变核心结论方向。

**关键挑战摘要**：
1. (B) 信息量等价假设——最需要补充的方法学讨论
2. (A) F1 粗粒度假说——建议补充 precision/recall 拆分
3. (C) 推广边界——需更清晰的限制声明

**审查后修正（已确认闭合）**：
1. 补充了天花板类别的逐类样本分布和 ceiling 阈值
2. 限制定了结论推广范围声明（GPT-5.5 中文编码任务内，temp=0）
3. 明确标注了 temp=0 选择和方法学限制
4. 补充了讨论：层级化 vs 扁平描述的信息量等价性问题

---

## 案例要点

1. **评推理不评结论**：D1/D3-D6 评的是"推理过程"，不是"结论对不对"——D4=3/5 是因为覆盖不完整，不是因为结论错误
2. **D2 先验证事实**：审查者先检查数据准确性——temp=0 和 n=4 的事实正确，但方法学后果（可推广性）留给 D3/D4 处理
3. **挑战使问题显式化**：(A)-(E) 的发现均在步骤二 D4 的 3/5 中已隐含，挑战让它们从"整体印象"变成了"具体可修复项"
4. **TOOL-LIMITED 而非强行判断**：审查者无法运行原始脚本，标注受限而非降级置信度
5. **阴性结果也有效**：A1 实验结论为阴性（Δ=0.000），但实验本身方法合格、报告诚实——Minor 而非 Major/Discard

关键教训：**单一后端的自我审查无法替代异后端独立审查**——审查者（GPT-5.5）在 D4 覆盖面和 (B) 信息量等价假设上发现了作者（DeepSeek-V4-Pro）在第 1-2 轮自审中未识别的问题。这恰好验证了 SOP §2.1 的核心主张。
